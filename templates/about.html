About the AI Model

A detailed look at the dataset, feature engineering, model architecture, and evaluation metrics powering this platform.

🎯

Platform Overview

The AI Salary Intelligence Platform uses a supervised machine learning model trained on real-world salary data to predict expected annual compensation based on a professional's profile. The model analyses 9 input dimensions, transforms them through a 42-feature engineering pipeline, and produces a salary estimate in under 100ms.

This is not a lookup table or a hardcoded formula — every prediction is computed from real statistical relationships learned from {{ metrics.get('total_records', '250,000') | int | format_number }} salary records.

Features Used

The model takes 9 raw professional attributes as input and transforms them into {{ metrics.get('num_features', 42) }} engineered features through one-hot encoding, scaling, and interaction creation:

🌍 Location
🏠 Remote Work
🏅 Certifications

Feature Engineering Pipeline

  • Categorical Encoding: Job Title, Education Level, Industry, Company Size, Location and Remote Work are one-hot encoded, expanding 6 categorical columns into 33 binary features.
  • Numerical Features: Experience Years, Skills Count and Certifications are kept as-is and standard-scaled.
  • Scaling: All 42 features are normalized using StandardScaler to zero mean and unit variance, ensuring the regularized regression model is not biased by feature magnitude.

Model Architecture

1

Raw Input (9 features)

User submits professional profile via the prediction form.

↓
2

ColumnTransformer Preprocessor

One-hot encodes categorical columns. Passes numerical columns through. Output: 42 features.

↓
3

StandardScaler

Normalizes all 42 features to zero-mean, unit-variance. Prevents scale dominance.

↓
4

Ridge Regression (L2 Regularized)

Linear model with L2 penalty to prevent overfitting on high-dimensional one-hot encoded data. Outputs continuous salary estimate.

↓
5

Predicted Annual Salary (₹)

Final output: estimated annual compensation rounded to 2 decimal places.

Explainability Method

Because Ridge Regression is a linear model, every prediction is fully decomposable as a weighted sum of feature contributions:

Salary = Intercept + Σ (coeffi × scaled_featurei)

Each feature's contribution in rupees is computed as coefficient × scaled input value. Features are then grouped into the 9 original input categories (Job Title, Experience, etc.) and ranked by their total contribution.

  • Positive factors push the prediction above the baseline intercept.
  • Negative factors reduce the prediction below the baseline.
  • This approach is equivalent to linear SHAP values and is mathematically exact — not an approximation.
